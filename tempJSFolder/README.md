# TODO List

## Основна задача

Вашата задача е да имплементирате JavaScript логиката, която "съживява" приложението. Предоставени са ви `index.html` и `style.css`. **Не трябва да ги променяте.**

Трябва да напишете своя код във файла `script.js`, като следвате коментарите и инструкциите в него.

## Функционалности на приложението

Готовото приложение включва следните функционалности:

1.  **Добавяне на задачи**:
    *   Въведете текст в полето "Добави нова задача...".
    *   Опционално, изберете крайна дата от полето за дата.
    *   Натиснете бутона "Добави" или клавиш "Enter".
    *   Всеки добавен елемент в списъка (`<li>`) трябва да има следната структура:
        *   Един `<span>` елемент, който съдържа текста на задачата.
        *   Втори `<span>` елемент с CSS клас `task-date`, който се показва само ако е избрана дата.
        *   Два бутона: един за "Завърши" (или "Върни") и един за "Изтрий" с CSS клас `delete-btn`.

2.  **Валидация на въвеждането**:
    *   Задачите трябва да са с дължина поне 5 символа.
    *   Полето за въвеждане предоставя визуална обратна връзка в реално време чрез CSS класове:
        *   ` .invalid ` (добавя се към `#task-input`): Прави рамката на полето червена, когато текстът е по-къс от 5 символа.
        *   ` .valid ` (добавя се към `#task-input`): Прави рамката на полето зелена, когато текстът е 5 или повече символа.
    *   Бутонът "Добави" е неактивен (`disabled` атрибут), докато не се въведе валидна задача.

3.  **Списъци със задачи**:
    *   Задачите са разделени в два списъка: "Незавършени" и "Завършени".
    *   В заглавието на всеки списък се показва броя на задачите в него.

4.  **Управление на задачите**:
    *   **Завършване**: Натиснете бутона "Завърши" до дадена задача, за да я преместите в списъка със завършени.
    *   **Връщане**: Натиснете бутона "Върни" до завършена задача, за да я върнете обратно в списъка с незавършени.
    *   **Изтриване**: Натиснете бутона "Изтрий", за да премахнете задачата напълно.

5.  **Зареждане на примерни задачи**:
    *   Натиснете бутона "Зареди задачи", за да изчистите текущите списъци и да заредите примерни данни от локалния `tasks.json` файл.

## Тестване с Cypress

Проектът е конфигуриран с Cypress за end-to-end (E2E) тестване.

*   Тестовите сценарии се намират в `cypress/e2e/todo_spec.cy.js`.
*   Тестовете покриват всички основни функционалности на приложението.

## Как да стартирате проекта

За да работите по проекта, ви е необходим [Node.js](https://nodejs.org/) и `npm`.

1.  **Инсталирайте зависимостите**:
    ```bash
    npm install
    ```

2.  **Стартиране на локален сървър**:
    В един терминал изпълнете:
    ```bash
    npm start
    ```
    Това ще стартира уеб сървър на адрес `http://127.0.0.1:8080`.

3.  **Стартиране на Cypress тестовете**:
    В друг терминал изпълнете:
    ```bash
    npm test
    ```
    Това ще отвори тестовия панел на Cypress, откъдето можете да стартирате тестовете.
